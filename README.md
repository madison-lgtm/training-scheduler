# Dora Training Scheduler

一个给 Dora 和会员使用的轻量排课网页。会员提交自己某一周的可用时间、训练重点和地点偏好；Dora 在工作台里生成草案、调整冲突，并发布最终安排。

## 会员怎么用

1. 进入 `Member 入口`。
2. 第一次先填写名字和识别码，建议识别码用手机号后四位。
3. 设置默认上课时间、训练内容和地点偏好。
4. 如果某一周照常，点 `照默认提交这一周` 就能提交给 Dora。
5. 如果某一周要变，先用顶部左右箭头切换到那一周，再点 `这周临时改`。
6. Dora 发布最终安排后，会员回到同一页输入名字，就能看到自己的课。

识别码不是正式登录密码，只是帮助 Dora 和会员确认身份。会员忘记识别码时，Dora 可以在工作台里查到。

## Dora 怎么用

1. 进入 `Dora 工作台`，输入 PIN。
2. 默认 PIN 是 `2468`，进入后可以在 `本周操作` 里修改。
3. `查看安排` 先看当前已发布或草案课表。
4. `调整排课` 可以生成草案、移动学员、处理冲突、选择每天地点。
5. 手机上可以点学员名字，再点目标时间格移动；电脑上也可以直接拖拽。
6. 确认后点 `发布最终安排`，会员就能看到自己的课程。

## 学员笔记

网页不能直接读取 Apple Notes。Dora 可以把 Apple Notes 里的每个学员重点复制到 `复制 Apple Notes 到这里`，粘贴后会自动保存。Firebase 打开时，这些笔记会跟其他排课数据一起云端保存，方便下次排课时查看。

## 日历导出

`导出当前周日历` 会下载一个 `.ics` 文件。这个文件可以导入 Google Calendar 或 Apple Calendar。导出的内容只包含当前选中周。

## 云端同步

这个版本支持 Firebase Firestore。配置完成后，所有人打开同一个 GitHub Pages 网址会读取同一份云端数据。

没有 Firebase 配置时，页面会使用本地演示模式，只保存在当前浏览器里。

## 安全说明

这个版本是朋友小范围使用的轻量工具。Dora PIN 可以防止普通会员误进工作台，但它不是严格的账号权限系统。如果以后会员变多，建议升级为正式登录和后端权限。

## 排课规则

- 每节课最多 2 人
- 同一天只在一个地点上课
- 每天地点根据会员偏好给出建议，最终由 Dora 决定
- 学员优先安排到自己提交的可用时间
- 同一个学员不能在同一时间段被排两节课
- 训练内容不太适合拼课时，系统会标记为需要 Dora 看一下
- 无法自动解决时进入 `还没安排的申请`，由 Dora 手动处理
