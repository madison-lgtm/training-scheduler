# Training Scheduler MVP

这是第一版可运行 MVP，用来把核心流程跑通：

- 学员统一入口提交本周训练申请
- 教练输入 PIN 进入工作台
- 教练生成排课草案
- 教练可以拖动学员到不同时间段
- 教练发布最终安排
- 学员输入自己的名字查看已发布课程

## 打开方式

直接打开 `index.html`。

## 云端同步

现在这个版本已经预留 Firebase Firestore 同步：

1. 到 Firebase 建一个项目。
2. 开启 Firestore Database。
3. 在 Firebase 项目的 Web App 设置里复制配置。
4. 打开 `firebase-config.js`，把 `enabled` 改成 `true`，并填入配置。
5. 上传到 GitHub Pages 后，同一个网址就会读取同一份云端数据。

没有填 Firebase 配置时，页面会继续使用本地演示模式。

## 当前限制

教练 PIN 是第一版轻量保护，适合朋友小范围使用，主要防止普通学员误进教练工作台。严格意义上，如果要保证学员无法通过技术方式读取教练数据，后续需要加一个真正的后端权限层或正式登录。

## 第一版排课规则

- 每节课最多 2 人
- 同一天只建议一个地点
- 地点平票时由教练选择
- 学员只会被安排到自己提交的可用时间
- 同一个学员不能在同一时间段被排两节课
- 上肢 + 下肢默认不优先自动拼
- 全身力量 + 功能性需要教练确认
- 无法自动解决时进入待放入池，由教练手动调整
